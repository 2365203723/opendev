# Development Workflow

## Feature Implementation
1. Plan first (use planner agent for complex features)
2. TDD: Write tests first → implement → refactor
3. Code review after writing code
4. Commit with conventional commits

## Research Before Coding
- GitHub search for existing implementations
- Library docs via Context7
- Check package registries before writing utility code

## Quality Gates
- Test coverage ≥ 80%
- Codex dual-blind review for all submissions
- Security review for auth/payment/crypto code
