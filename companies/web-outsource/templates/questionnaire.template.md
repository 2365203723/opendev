# 问卷模板 · IntakeAnalyst 用

> IntakeAnalyst 填完后交给老板。老板勾选或自行改写。
> **原则**：不超过 10 题；每题都是是非 / 选择题；给默认值。

---

## 必答（不答则卡住）

### 1. 项目优先级（强制选一）
- [ ] 完成度第一（MVP 功能跑通即可）
- [ ] 美观第一（UI 必须精致）
- [ ] 性能第一（对接口延迟敏感）

### 2. 交付形式
- [ ] 本地跑（一条命令启动）
- [ ] 部署到客户自有服务器
- [ ] 部署到公有云（需告诉我哪家）
- [ ] 仅交源码 + 文档

### 3. 预算范围（token 消耗 or 项目报价）
- [ ] MVP（< $500）
- [ ] 标准（$500 ~ $5000）
- [ ] 大单（> $5000，走周报机制）

### 4. 截止日期
YYYY-MM-DD（今天、本周、本月、日期待定）

---

## 可选（不答走默认）

### 5. 目标用户数量级
- [ ] 个人 / 内部用（< 10）
- [ ] 小团队（10-100）
- [ ] 中规模（100-10000）
- [ ] 大规模（> 10000，需认真谈性能）

### 6. 数据持久化要求
- [ ] 内存 OK（重启清空）
- [ ] 本地文件 / SQLite
- [ ] 独立数据库服务（需告诉我 MySQL/Postgres/MongoDB）

### 7. 用户身份
- [ ] 无需登录
- [ ] 单一密码登录（内部工具）
- [ ] 完整注册登录系统

### 8. 视觉风格参照
- 参照网站 URL：___________________
- 或者：克制 / 活泼 / 商务 / 极简（选一）

### 9. 技术栈限制（客户有提到就传达，没提就"Architect 拍板"）
- [ ] Architect 自主（推荐）
- [ ] 必须用 Node.js
- [ ] 必须用 Python
- [ ] 必须用 <具体>

- [ ] 用户注册登录

- [ ] _________________
