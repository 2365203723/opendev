# Handoff 文件强制格式

> 版本 v1.0（2026-05-15）
> 所有 agent 写 handoff 时必须遵守此格式。下游 agent 发现缺字段可拒收。

## 文件命名

`doc/handoff/<from>-to-<to>.md`

示例：`ceo-to-pm.md`、`qa-to-security.md`、`reviewer-to-devops.md`

## 固定字段（6 个，缺任何一个 = 不合格）

```markdown
# <from> → <to> 交接

## 完成项
- 列出本阶段所有产出文件路径（相对于项目根）
- 每个文件一行

## 未完成项 / 遗留
- 列出未解决的问题（无则写"无"）

## 决策与理由
- 本阶段做的关键决策 + 为什么这么决定
- 无决策写"无"

## 给下一角色的具体动作
1. 第一件要做的事（动词开头）
2. ...

## 风险与警告
- 下游需要注意的坑（无则写"无"）

## 验收标准映射
| charter 条款 # | 本阶段覆盖情况 | 备注 |
|---|---|---|
| 1 | 已覆盖 / 部分覆盖 / 未涉及 | ... |
```

## 规则

1. **不可省略字段**：即使内容是"无"也要写出字段标题
2. **验收标准映射**：必须列出 charter 中所有条款编号，标注本阶段是否涉及
3. **具体动作**：必须是可执行的指令，不能是"请参考 design.md"这种模糊引用
4. **文件路径**：用相对路径（相对项目根），不用绝对路径
5. **无活可干时**：仍需写 handoff，完成项写"本阶段无需<角色>工作"，其余字段正常填

## 下游 agent 读取方式

```
1. 读 doc/handoff/<上游>-to-<自己>.md
2. 检查 6 个字段是否齐全
3. 从"给下一角色的具体动作"开始执行
4. 从"验收标准映射"确认自己要覆盖哪些条款
```
